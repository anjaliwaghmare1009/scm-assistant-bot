# Architecture Overview

The chatbot is built using a Retrieval-Augmented Generation (RAG) architecture.

Workflow:
1. Dataset ingestion
2. Supplier-level document generation using Python preprocessing
3. OpenAI embeddings generation
4. Storage in Pinecone vector database
5. Similarity-based retrieval
6. GPT-4o-mini answer generation

# Data Preprocessing

## Initial Problem

The original dataset contained purchase-order (PO) level records.

A single supplier appeared across multiple rows because every purchase order was stored as a separate transaction.

This created several issues:
- Duplicate supplier retrieval
- Inconsistent supplier counts
- Different answers for the same question
- Difficulty answering supplier-level analytical questions

## Solution

A Python preprocessing script was created to aggregate all purchase-order records belonging to the same supplier and generate supplier-level documents.

---

## Supporting Files

### supplier_aggregation_script.py

This Python script was created to solve the issue of duplicate supplier records in the original purchase-order dataset.

The script aggregates multiple purchase-order records belonging to the same Supplier_ID and generates a consolidated supplier profile containing supplier metrics, risk information, certifications, and disruption data.

### supplier_profiles.txt

This file contains the supplier-level documents generated by the aggregation script.

These documents were indexed into Pinecone and used as the primary knowledge source for the RAG chatbot.
